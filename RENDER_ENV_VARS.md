# Render Environment Variables Setup

## Before Deploying Blueprint

When deploying the Blueprint, you'll need to enter these environment variables:

### Backend Service (`finance-tracker-backend`)

**FRONTEND_URL:**
```
https://finance-tracker-frontend.onrender.com
```
*(Update this after frontend is deployed if the URL is different)*

### Frontend Static Site (`finance-tracker-frontend`)

**VITE_API_URL:**
```
https://finance-tracker-backend.onrender.com/api
```
*(This should be correct - backend URL + /api)*

## After Deployment

1. **Check the actual URLs** in your Render dashboard:
   - Backend URL will be shown in the backend service
   - Frontend URL will be shown in the frontend service

2. **Update environment variables** if URLs differ:
   - Go to backend service → Environment tab
   - Update `FRONTEND_URL` with the actual frontend URL
   - Go to frontend service → Environment tab  
   - Update `VITE_API_URL` with the actual backend URL + `/api`

3. **Restart services** after updating environment variables

## Notes

- The database URL (`DATABASE_URL`) is automatically set by Render
- JWT secrets (`JWT_SECRET`, `JWT_REFRESH_SECRET`) are auto-generated by Render
- `ENCRYPTION_KEY` is auto-generated by Render
- You can regenerate secrets later if needed in the Render dashboard
